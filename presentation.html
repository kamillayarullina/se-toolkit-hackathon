📋 QueueLess

A modern TA office-hour booking system with admin controls, notifications & full audit logging

Kamil Iarullina

k.iarullina@innopolis.university

Group B2301

Context

End Users

  • Students at Innopolis University who need to book TA consultations
  • Teaching Assistants who manage their availability
  • Administrators who configure schedules and monitor activity

The Problem

  • No centralized system to find and book TA office hours
  • TAs can't define availability or block holidays
  • No notifications or audit trail for bookings

QueueLess solves this with a self-hosted web app — browse, book, and manage TA slots in one place.

Implementation

Tech Stack

  • Backend: Node.js + Express
  • Database: SQLite3
  • Frontend: Vanilla HTML/CSS/JS
  • Notifications: Nodemailer (Gmail) + Telegram Bot API
  • Deployment: Docker + Docker Compose

Version 1 → Version 2

  • V1: Basic booking with overlap detection, single TA
  • V2: Week/month calendar, admin auth, available hours, blocked dates, email/Telegram notifications, booking history log, password change

TA Feedback Addressed

  • ✅ Admin can define available hours per day
  • ✅ Admin can block dates (holidays)
  • ✅ Email/Telegram notifications on confirm & cancel
  • ✅ Booking history log (who, when, what action)
  • ✅ Admin authentication for all management actions
